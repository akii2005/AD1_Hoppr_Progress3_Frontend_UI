# Progress 3 Video Script

## 1. Opening

Hello Dr., this is the Progress 3 frontend UI demonstration for our project, Hoppr / UTM Ride Hailing App. The app is designed for the UTM community to provide verified campus ride booking, parcel delivery, food delivery, driver management and admin monitoring.

## 2. What has been completed

For Progress 3, we completed a working high-fidelity frontend UI prototype using HTML, CSS and JavaScript. The prototype includes the main screens from our Progress 1 and Progress 2 documents.

## 3. Authentication and profile

First, the system shows the login screen. Users can log in using a verified UTM email. The prototype also includes registration, email verification and reset password. After login, users can view and edit their profile, update phone number and faculty details, and change settings such as dark mode and notifications.

## 4. Passenger ride flow

The passenger can book a ride by selecting pickup and drop-off locations inside UTM, choosing vehicle type and service tier, viewing a fare estimate, confirming the ride, tracking the assigned driver, and recording payment.

## 5. Delivery service flow

The delivery module includes parcel delivery and food delivery. For parcels, the user enters pickup, drop-off and parcel details. For food delivery, the user selects a campus vendor and enters the food order. Both flows include tracking status and delivery confirmation.

## 6. Driver flow

The driver dashboard allows a student driver to go online or offline, view available ride, parcel and food jobs, accept or decline jobs, complete trips and view earnings.

## 7. Admin flow

The admin screen allows the admin to verify student accounts, monitor ride and delivery activity, review complaints and suspend accounts if needed.

## 8. Trello and GitHub

We also updated the project management part by preparing the source code for GitHub and including the module-to-frontend-script mapping table in the README file. The GitHub link and video link will be placed in the Progress 3 report front page.

## 9. Closing

That completes our Progress 3 frontend UI prototype demonstration. Thank you.
